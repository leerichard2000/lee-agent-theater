# Código de Conduta

Lee Agent Theater é um projeto open-source e depende de interações
saudáveis entre quem contribui. Este documento descreve, de forma curta
e direta, o que esperamos de quem participa das discussões, issues e
pull requests.

## Nossos valores

- **Respeito mútuo** em todas as interações — issues, PRs, discussões
  e comentários de código.
- **Construtivo antes de crítico.** Feedback técnico, não pessoal:
  falamos sobre código, ideias e decisões — nunca sobre a pessoa por
  trás delas.
- **Sem assédio, discriminação ou comportamento tóxico** de qualquer
  forma, incluindo linguagem ofensiva, trolling intencional, ataques
  pessoais, ou invasão de privacidade.
- **Presuma boa-fé.** Em caso de dúvida, pergunte antes de acusar.
  Contribuidores estão aqui voluntariamente; reviews e comentários
  devem tratar isso com a consideração que merece.
- **Seja paciente.** Nem todo mundo tem o mesmo contexto. Explicações
  claras em PRs e issues ajudam mais do que respostas curtas e secas.

## Escopo

Este código se aplica a todos os espaços do projeto: repositório no
GitHub (issues, PRs, discussions, reviews), comunicações diretas entre
contribuidores sobre o projeto, e qualquer espaço público onde alguém
esteja representando o Lee Agent Theater.

## Reportar problemas

Comportamento inadequado pode ser reportado diretamente para o autor
original do projeto, **Lee Richard**, pelo e-mail registrado no
[`package.json`](package.json). Todos os reports serão tratados com
confidencialidade. Não é necessário ter certeza absoluta para reportar —
se algo parecer errado, vale mandar.

## Consequências

Comportamento que viola este código pode resultar em:

1. Aviso privado, com indicação específica do que foi inadequado.
2. Remoção de comentários, commits, issues ou PRs problemáticos.
3. Banimento temporário ou permanente de participação no projeto, a
   critério do mantenedor.
